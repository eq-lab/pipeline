# Pipeline User Docs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a 19-page GitHub Pages site at `docs/user-docs/` that introduces Pipeline to qualified on-chain lenders, explains how the protocol earns yield, discloses risk honestly, and documents the split-rail security architecture — matching the visual language of `overview.html`.

**Architecture:** Jekyll site under `docs/user-docs/`, deployed to `gh-pages` via a GitHub Actions workflow. Nine architecture diagrams as inline SVG (hand-authored to match overview.html's Tokyo-Night palette), three charts generated by a committed Python matplotlib script. Content is written page-by-page through a Writer → Reviewer → Commit loop; each flow diagram ships with a numbered step-by-step walkthrough in prose.

**Tech Stack:** Jekyll 4.x, Liquid templates, GitHub Actions, Python 3.11 + matplotlib (for charts), inline SVG (for diagrams), plain CSS.

**Source of truth:** `docs/superpowers/specs/2026-04-23-user-docs-design.md`. Working tree: `/tmp/pipeline-repo/` on branch `docs/update-specs-v2.3`.

**Reviewer agent template** (referenced as `{REVIEWER(path)}`):

```
Agent: general-purpose, foreground, ~300 words output.
Prompt:
  Review the markdown page at {path} as if you were an institutional reader landing
  cold on Pipeline's docs. Return a JSON object with fields:
    { pass: bool, diffs: [<specific edits>], notes: [<non-blocking notes>] }

  Check each of the following and fail the page if any check fails:
    1. Plain English: sentences under 25 words, active voice, no filler adjectives
       (leverage, seamless, world-class, institutional-grade, cutting-edge, robust,
       innovative, best-in-class). No AI self-reference.
    2. Retired v2.3 terms absent: MINT_ATTESTOR, LOAN_MANAGER, loan_manager,
       EmergencyRevoker, revokeAll, HSM, off-chain wire, F-8 pre-fund,
       adjustRecoveryRateDown, convertSharesAtShutdown. Grep confirms.
    3. Every number printed on the page matches the approved set in the design spec
       (§ "Numbers to quote"). List any number that doesn't match with its line.
    4. Disclosure posture: any sales claim about yield, caps, or safety is followed
       in the same section by a stated constraint or risk. Flag any claim that
       isn't.
    5. Internal consistency: statements about architecture match
       /tmp/pipeline-repo/docs/product-specs/*.md. Flag any contradiction with file
       and line reference.
    6. Flow-diagram walkthroughs: if the page embeds a diagram whose ID starts with
       D2, D3, D4, D5, D6, D7, or D9, it must carry a numbered step-by-step
       walkthrough using the same labels that appear as nodes in the SVG. Reject
       the page if this is missing.
    7. No placeholders (TBD, TODO, coming soon) except where the design spec lists
       them as explicit "Open placeholders".

  If pass=false, return diffs concrete enough to apply directly.
```

**Glossary append rule.** Every Writer run ends by appending any term it used but did not define to the `<!-- pending-glossary -->` comment block at the bottom of `docs/user-docs/glossary.md`. Task 38 consolidates.

### Task 5: GitHub Actions workflow

**Files:** Create: `.github/workflows/user-docs.yml`

- [ ] **Step 1: Write the workflow**

```yaml
name: Deploy user-docs to GitHub Pages

on:
  push:
    branches: [docs/update-specs-v2.3, main]
    paths: [docs/user-docs/**]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: docs/user-docs
    steps:
      - uses: actions/checkout@v4
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.3'
          bundler-cache: true
          working-directory: docs/user-docs
      - name: Install Jekyll
        run: |
          gem install bundler jekyll
          bundle init || true
          cat > Gemfile <<'EOF'
          source "https://rubygems.org"
          gem "jekyll", "~> 4.3"
          gem "jekyll-feed", "~> 0.17"
          gem "jekyll-sitemap", "~> 1.4"
          gem "jekyll-redirect-from", "~> 0.16"
          EOF
          bundle install
      - name: Build site
        run: bundle exec jekyll build
        env:
          JEKYLL_ENV: production
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/user-docs/_site

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

def c1_reserve_composition():
    """Reserve composition pie — at the 15% target USDC buffer (design spec)."""
